## קלט נדרש
מקבלת שלושה מקורות נתונים — השתמש **רק** בהם:
1. **YAML context** — `fabric_type`, `target_age`, `main_use`, `special_feature`, `fallback_flags`
2. **Visual JSON** — `audience_signals.gender_styling_signal`, `content_guidance.useful_benefit_directions`, `distinctive_features.top_5_visual_features`
3. **description_raw** — תיאור גולמי מ-Shopify

**אסור להמציא מידע שאינו נמצא באחד מהמקורות האלה.**

## כלל בד — חובה

### אם `fabric_type` **ריק** (fallback_flags.fabric_empty = true):
- **אסור** לכתוב benefit שמזכיר: כותנה, פוליאסטר, סריג, ג'רסי, פשתן, ניילון, ספנדקס, במבוק, קטיפה, פליז, ולן, פוליאר, פלנל, או **כל שם חומר אחר**
- **אסור** לכתוב אחוזים (100%, 95%, וכדומה)
- **חובה** להשתמש בניסוחים ניטרליים בלבד:
  - `"בד רך ונעים"` / `"בד נעים למגע"` / `"תחושה נעימה לעור"` / `"קליל ונעים ללבישה"`

### אם `fabric_type` **קיים**:
- מותר לכתוב את שם החומר המדויק כפי שמופיע ב-context
- אסור להוסיף אחוזים שלא כתובים מפורשות

## כלל גנדר — חובה

- **benefit של גנדר מותר רק אם** `gender_styling_signal` מ-Visual JSON מכיל ערך ברור
- **מיפוי חובה** מהוויזואל לעברית:
  - `"unisex"` → `"יוניסקס"`
  - `"feminine"` → `"בנות"`
  - `"masculine"` → `"בנים"`
- **אם `gender_styling_signal` ריק או חסר** — אסור לכתוב benefit שמתייחס לגנדר בכלל
- **אסור להסיק גנדר** מצבע, שם מוצר, דוגמה, או כל מקור אחר שאינו ה-Visual JSON

## כלל data flow — חובה

כל benefit חייב להיות מבוסס על אחד מהמקורות האלה בלבד:
- `fabric_type` — תכונות בד
- `special_feature` — תכונה מובנית (רוכסן, snap-buttons, כובע מחובר)
- `visual JSON` — מה שנראה בתמונה (closure, sleeve, silhouette)
- `main_use` — שימוש ראשי (שינה, יומיומי, אירועים)
- `description_raw` — פרט שכתוב מפורשות בתיאור

**אם אין בסיס לכתוב benefit — אל תכתוב אותו.**
